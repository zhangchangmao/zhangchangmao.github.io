核心解析

背景： 通用大模型在特定领域不够专业，且处理复杂任务时十分困难。Skills 充当了“桥梁”，将人类经验、知识和流程进行封装

原理： 采用 Agent + Skills + Computer 架构。智能体在虚拟环境中通过“使用计算机”的能力，自动读取并运行文件系统中的技能脚本。

结构： 每个 Skill 文件夹包含一个核心 SKILL.md（定义流程与描述）以及存放示例、模板和代码的资源文件夹。

Skills vs. MCP

Skills 技能

  • 定位： 知识共享与流程封装。
  • 形式： 基于 Markdown，结构简单，人人可创。
  • 核心优势： 节省 Token，复用性强，易于传播。
  • 适用场景： 固化经验、标准化操作、最佳实践共享。

MCP (Model Context Protocol)

  • 定位： 功能扩展与外部集成。
  • 形式： 侧重连接API、数据库或特定工具，配置相对复杂。
  • 核心优势： 突破模型边界，访问实时/私有数据，执行具体操作。
  • 适用场景： 需要与外部系统交互、获取实时信息的任务。

结论： Skills 与 MCP 是互补共生的协作关系。Skills 负责封装“怎么做”的知识，MCP 负责提供“连接什么”的能力。
